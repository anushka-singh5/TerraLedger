How it works — end to end

Verification gates the mint. Credits are fungible. Retirement is permanent and tracked per project.

1
Submit + AI verify 5 modules · score ≥ 70
Developer submits GPS polygon, deed PDF and tonnage claim. Five AI modules run in sequence: GPS duplicate detector, deed forensics (OCR + ELA forgery check), anomaly model trained on Verra registry data, real NASA FIRMS satellite check and a Llama-3 plain-English audit. GPS overlap >20% or forged deed = instant hard-block, logged on-chain forever. Score below 70 = no mint.
2
Oracle mints TLCERT + TCC soulbound proof + fungible credits
On pass, the M-of-N oracle mints two things directly to the developer's wallet: a TLCERT soulbound NFT (non-transferable verification proof, bound to the project ID forever) and TCC ERC-20 tokens (1 TCC = 1 tonne CO₂, tradeable). TLCERT cannot be sold — it stays with the developer as permanent on-chain proof the project cleared all AI checks.
3
List & trade TCC QUSDC · per-tonne pricing · projectId stored on-chain
Developer lists TCC at a per-tonne QUSDC price on the marketplace. Every listing stores the source projectId on-chain — buyers always know which verified project they're buying credits from, even after credits change hands. No anonymous TCC floating around the market.
4
Retire TCC → TLRET certificate permanent offset + PDF download
Any TCC holder calls retire(). Tokens burn permanently against the correct project ID. A TLRET soulbound certificate auto-mints to the retiree's wallet (amount, projectId and block timestamp stored on-chain), and a PDF retirement certificate is instantly downloaded — with tx hash and full details. One retire call = one certificate. No double-counting possible.
5
TLCERT auto-offsets no button · no re-minting · tamper-proof
The contract tracks TCC issued vs. retired per project ID. When every TCC from a project is burned, the TLCERT automatically flips to Fully Offset — no manual action, no admin call. Since the project ID is locked at mint and GPS coordinates are registered on-chain, the same land parcel cannot generate new credits once fully offset.
